## What this project is

Epigenetic regulation of tail regeneration in the slug *Deroceras laeve*. We have WGBS (methylation) and RNA-seq (expression) data and need to write a Science Advances paper asking: **does DNA methylation regulate gene expression during invertebrate regeneration?**

All code must be in R. Assume NOTHING is correct until you run it and see the output.

## Organism

- **Species**: *Deroceras laeve* (gray field slug, gastropod mollusk)
- **Non-model invertebrate** — no UCSC, no Ensembl, no CpG islands, no mammalian assumptions
- **Genome**: custom assembly, 1.78 Gb, 31 chromosomes (chr1–chr31), 42.99% GC. ALWAYS filter out HiC_scaffold_* before analysis.
- **Annotation**: EviAnn GFF (see data paths below)

## Experimental design

### WGBS (methylation)
- 2 control (C1, C2) vs 2 amputated (A1, A2) tail samples
- Bismark CpG_report.txt files (7 columns: chr, pos, strand, count_meth, count_unmeth, context, trinuc)
- DSS for differential methylation (NOT MethylKit)

### RNA-seq (expression)
- Multi-tissue: tail, eye, bodywall, head, juvenile, ovotestis
- Multi-condition: control, amputated, irradiated, fungicide
- Haven't been done test to see if there are outliers
- DESeq2 for DE. WGCNA for co-expression modules.

## Paper structure — Science Advances

### 3-act narrative

**Act 1: The genome** — What does D. laeve look like before we touch methylation?
- 1.1 Methylation machinery (DNMT1/DNMT3/TET/MBD identification + expression)
- 1.2 CpG O/E ratio genome-wide and per region (compute from genome FASTA)
- 1.3 Promoter CpG architecture (Weber classification — data exists in promoter_cpg/)

**Act 2: The methylome at rest** — Baseline patterns, no differential analysis yet
- 2.1 Global methylation landscape (bimodal? where are methylated CpGs?)
- 2.2 Gene body methylation vs expression (is it positive like other invertebrates?)
- 2.3 CpG density profile around TSS (replaces "shore" analysis — no CpG islands exist)
- 2.4 TE methylation at baseline

**Act 3: The methylome after amputation** — What changes, does it matter?
- 3.1 DMPs and DMRs (how many, where, what direction)
- 3.2 Do methylation changes predict expression changes? (test with MI, correlation, ML)
- 3.3 Co-expression modules reveal hidden structure (WGCNA)
- 3.4 Hub gene behavior
- 3.5 Case study: Sox19a intergenic regulation

### Figure plan (≤10 main figures)

| Fig | Title | Act | Data source |
|-----|-------|-----|-------------|
| 1 | Methylation toolkit + expression heatmap | 1 | annotations + counts |
| 2 | CpG landscape: O/E, density, promoter classes | 1 | genome FASTA + GFF + promoter_cpg/ |

| 4 | Baseline meth vs expression + TSS profile | 2 | bsseq + VST counts |

| 9 | WGCNA modules resolve the paradox | 3 | WGCNA + DMP data |
| 10 | Sox19a case study | 3 | DMP + expression + GFF |

### Key questions the paper must answer (with computation, not assumptions)

1. Does D. laeve have methylation machinery? → grep annotations, show expression
2. Where are CpGs? → count from genome FASTA per region
3. Where is methylation? → summarize bsseq beta values per region
4. Does baseline methylation correlate with expression? → Spearman(mean_beta, expression)
5. What changes during regeneration? → DMPs/DMRs from DSS
6. Do changes predict expression? → compute MI from contingency table, test with permutation
7. If not globally, then where? → per-module correlations from WGCNA
8. What's the mechanism? → intergenic regulation, binary switch evidence

## CRITICAL rules

1. **ALL numbers must come from running code.** If you haven't run it, you don't know it. Never paste a number from a comment or markdown file into the paper.
2. **This is NOT a mammal.** No CpG islands. No promoter silencing paradigm. No DNMT3. Do not apply mammalian logic.
3. **ALL code in R.** No Python for analysis.
4. **Filter scaffolds first.** `keep_chr <- paste0("chr", 1:31)` at the top of every script.
5. **Use established Bioconductor packages.** GenomicRanges, rtracklayer, bsseq, DSS, DESeq2, WGCNA, genomation. Don't reinvent.
6. **txdbmaker is NOT available** in Claude Code R env. Use `rtracklayer::import()` + GenomicRanges instead of `makeTxDbFromGFF()`.
7. **Mutual information must be computed properly.** Build a 2×2 contingency table (hyper/hypo × up/down), compute MI from joint vs marginal probabilities, test significance with permutation. Use the `infotheo` package or write the formula explicitly. Show your work.
8. **Every figure: PDF (cairo_pdf) + PNG (300 DPI).** Dual save always.
9. **Think before coding.** What question? What expected result? What test? Plan the analysis design first.
10. **Memory management.** The genome is 1.78 Gb — `overlapsAny()` on 45M CpG GRanges can OOM. Use per-chromosome splitting (`split(cpg_all, seqnames(cpg_all))`) or sequence extraction (`dinucleotideFrequency`) instead. Call `rm()` + `gc()` after large objects are no longer needed.

## Batch workflow rules

1. **One batch = one script = all output.** Each `scripts/batchNN/*.R` script generates every plot (PDF + PNG) AND the HTML report in a single run. Never split a batch across multiple scripts or patch files.
2. **The script is the single source of truth.** If a plot or report needs updating, edit the batch script and rerun it — don't manually edit the HTML or run side scripts.
3. **Every plot must appear in the report.** If the script creates a PNG, the HTML report section must have a matching `<img>` tag. No orphan plots.
4. **PDF lock workaround (WSL/Windows).** Windows can lock PDFs open in a viewer. Scripts should save PNG first (always works), then write PDF to `TMPDIR` and `file.copy()` to the final location. If copy fails, warn but don't crash:
   ```r
   tmp_pdf <- file.path(Sys.getenv("TMPDIR", "/tmp"), "plot_name.pdf")
   cairo_pdf(tmp_pdf, width = w, height = h); print(p); dev.off()
   ok <- file.copy(tmp_pdf, final_path, overwrite = TRUE)
   if (!ok) cat("WARNING: PDF locked, saved to:", tmp_pdf, "\n")
   ```
5. **Don't write patch scripts.** If something is missing or outdated, fix the batch script itself and rerun. The batch script must be re-runnable end-to-end at any time.

```r
                  overwrite = TRUE)
  if (!ok) cat("  WARNING: PDF locked, saved to:", tmp_pdf, "\n")
  cat("  Saved:", name, "\n")
}

# --- 4. LOAD DATA (use RDS cache when possible) ---

gff    <- readRDS(file.path(cache_dir, "gff_chr1_31.rds"))

# --- 5. ANALYSIS ---
# ... computations ...

# --- 6. PLOTS ---
# One save_plot() call per figure. Name must match what goes in the report.

# --- 7. HTML REPORT ---
# sink() to out_dir/batchXX_report.html
# Every plot must have an <img src="png/name.png"> tag.
# sink() closes report.

```

### Key procedures

1. **Loading large data.** Always use RDS cache (`readRDS`) instead of re-parsing FASTA/GFF/TSV. Cache files live in `genome/cache/`. If a cache doesn't exist, parse once and `saveRDS()`.
2. **Genome-wide CpG operations.** Two approaches, pick by context:
   - **countOverlaps / overlapsAny**: fast for moderate-size region sets. Pre-split CpG GRanges by chromosome (`split(cpg_all, seqnames(cpg_all))`) to avoid 45M-row operations.
   - **dinucleotideFrequency on Views**: memory-safe for TE/region analysis. Extract sequences per chromosome, count CG dinucleotides. No need for a full CpG GRanges object.
3. **DESeq2 workflow.** Build sample table → `DESeqDataSetFromHTSeqCount` → filter low counts (`rowSums > 1`) → `DESeq()` → `varianceStabilizingTransformation(blind=FALSE)` for visualization. Use `assay(vsd)` for the normalized matrix.
4. **Per-chromosome loops.** For any operation touching the full genome, loop over chromosomes and `rm(seqs); gc()` after each to keep memory bounded:
   ```r
   for (chr_name in names(genome)) {
     chr_reg <- regions[seqnames(regions) == chr_name]
     v <- Views(genome[[chr_name]], start = start(chr_reg), end = end(chr_reg))
     seqs <- as(v, "DNAStringSet")
     # ... compute ...
     rm(seqs); gc(verbose = FALSE)
   }
   ```
5. **Distribution plots.** When showing bar charts of summary stats (e.g., O/E by region), always also show the per-chromosome density distributions. This shows variance, not just means. Use `geom_density(alpha = 0.3)` with reference lines for genome average and expected value.
6. **Heatmaps (pheatmap).** Use `pheatmap(..., silent = TRUE)` to capture the grob, then draw with `grid::grid.draw(p$gtable)` inside a device. Z-score rows with `t(scale(t(mat)))`. Add `annotation_row` for gene categories.
7. **HTML reports.** Use `sink(report_path)` / `sink()` to write HTML. Build tables row-by-row with `cat()`. Embed plots as `<img src="png/name.png">`. Every plot the script creates MUST appear in the report.
8. **Adding a new batch.** Create `scripts/batchNN/`, create `results/batchNN/`. Write one R script following the skeleton above. After running successfully, update the "Completed batches" table in this file.
9. **JASPAR motif scanning protocol.** This is the established procedure for TF motif annotation:
   - **Setup**: Set `XDG_CACHE_HOME` env var BEFORE loading any Bioconductor packages (JASPAR2024 uses BiocFileCache).
   - **JASPAR database**: Use `JASPAR2024()` to get SQLite path, then `getMatrixSet(sqlite_path, list(collection = "CORE", all_versions = FALSE))`. Filter metazoa via `tags(x)$tax_group %in% c("insects", "nematodes", "urochordates", "vertebrates")`. No mollusk-specific motifs exist — insect/nematode proxy.
   - **Promoter extraction**: 2 kb upstream of gene TSS. Cache as `genome/cache/promoters_2kb.rds`. Handle strand: `+` strand → `[start-2000, start-1]`, `-` strand → `[end+1, end+2000]`. Filter invalid promoters (`start > end` at chromosome boundaries).
   - **Motif scan (fast, local)**: `matchMotifs(pfms, sequences, out="scores", p.cutoff=5e-5)` → boolean hit matrix (~45 sec, runs locally). Extract with `motifMatches()`.
   - **Motif positions (slow, cluster)**: `matchMotifs(pfms, sequences, out="positions", p.cutoff=5e-5)` → exact genomic coordinates. Needs ~32 GB RAM. Run on cluster, save as compressed TSV.
   - **Coordinate conversion**: Relative motif positions within promoter sequences must be converted to genomic coordinates. `+` strand: `genomic = prom_start + relative - 1`. `-` strand: `genomic_start = prom_end - rel_end + 1`. Flip motif strand for `-` strand promoters.
   - **TF family classification**: Use keyword matching on EviAnn gene names + descriptions. Map TF families to JASPAR family names using exact JASPAR 2024 naming (e.g., "Three-zinc finger Kruppel-related", "FOX", "Ets-related"). Cross-reference protein predictions vs genome motifs for confirmation.
   - **Extra conda packages needed**: `bioconductor-jaspar2024`, `bioconductor-motifmatchr`, `bioconductor-universalmotif`, `r-rsqlite`

### How to run a cluster job

1. **Write the script** in `cluster/scripts/`. Make it self-contained (loads its own data from RDS cache, no dependencies on local R session). Use `PROJECT_ROOT` env var for paths.
2. **Write a SLURM file** in `cluster/scripts/`. Request adequate RAM (32 GB typical). Set `--partition` to your cluster's queue.
3. **Copy to cluster**: the whole STANDBY project (or at minimum `genome/cache/*.rds` + `cluster/`).
4. **Submit**: `sbatch cluster/scripts/job_name.slurm`
5. **Copy results back**: `cluster/results/*` → `results/batchNN/`

### Rules

- Cluster scripts are **extraction jobs only** — they produce data files (TSV, RDS), not plots or reports. Plots and reports are always generated locally by the batch script.
- After copying results back, the batch script should detect and use them (e.g., check if `motif_hits_annotated.tsv.gz` exists before attempting the scan locally).
- SLURM logs go to `cluster/results/` alongside the output.
